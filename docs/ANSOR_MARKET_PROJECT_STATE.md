## 2026-07-02 KakaoMap Address Search and Local Upload Cleanup Bugfix Checkpoint

### Completed

- Fixed Telegram Mini App order detail KakaoMap opening:
  - removed coordinate-based KakaoMap URL generation from the order detail button
  - button now opens Kakao mobile search URL:
    - `https://m.map.kakao.com/scheme/search?q=${encodeURIComponent(addressText)}`

    2. `jibunAddressSnapshot`

    4. `buildingNameSnapshot`

- Removed active coordinate-only/Google map links from API/web/admin source searches.
- Added backend local upload cleanup:
  - `StorageService.deleteUploadedFileIfExists`
  - `StorageService.resolveUploadFilePath`
  - local files are deleted only when the DB value resolves safely under the API `uploads` directory

  - path traversal and random absolute paths are rejected
  - missing files are non-fatal
  - delete failures are logged as warnings and do not crash product delete
- Wired product image cleanup into product admin flows:
  - product delete removes local uploaded images after the DB delete succeeds
  - product update removes old local uploaded images that were replaced by the new `images` array
- Updated the requirements doc so KakaoMap links are address-search-only and product local upload cleanup is part of the project contract.
